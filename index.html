Frequently asked questions

What does an AI agent to build websites actually do?

It researches requirements, generates site structure and code, operates real tools (browser, GitHub, hosting dashboards) to deploy, and then iterates updates using the same computer steps.

How is this different from AI website builders?

Website builders generate pages inside their own tools. A computer-using agent works across real apps you already use—GitHub, hosting, analytics—by operating a browser and desktop steps.

Where does the computer-use cache help?

When the agent repeats the same login, deploy, or publish steps, Super can reuse a computer-use cache so those workflows get faster and cheaper over time.

Can Super push to GitHub and publish automatically?

Super can assist with commits, pull requests, and publishing by operating the GitHub UI and hosting dashboards, with user confirmation for sensitive actions.

Is this safe for production websites?

Yes, when used intentionally. Actions like deployments or DNS changes should require review and confirmation, and Super is designed for scoped, auditable computer use.
